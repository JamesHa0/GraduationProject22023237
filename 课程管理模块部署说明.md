# 课程管理模块部署说明

## 模块概述

课程管理模块包含以下功能：
1. 课程设置：支持学院根据培养方案设置课程信息，包括课程名称、学分、学时、授课教师等
2. 学生选课管理：教学秘书添加课程后，研究生可通过系统自主选课，系统提供课程介绍、教师评价等信息辅助学生选课
3. 成绩管理：导师可在线查看学生，批量录入学生成绩（后续可修改），学生和导师可实时查询成绩信息

## 部署步骤

### 1. 数据库配置
1. 确保数据库中已经创建了 `course`、`course_selection`、`score` 表（你提供的 SQL 脚本已经包含了这些表）
2. 执行 `course_menu.sql` 脚本，在菜单表中添加课程管理相关菜单
```sql
source /data/app/GraduationProject22023237/course_menu.sql;
```

### 2. 后端部署
1. 编译后端项目
```bash
cd /data/app/GraduationProject22023237/gp22023237-server
mvn clean install
```
2. 启动后端服务
```bash
java -jar target/gp22023237-server-*.jar
```

### 3. 前端部署
1. 安装前端依赖
```bash
cd /data/app/GraduationProject22023237/gp22023237-ui
npm install
```
2. 构建前端项目
```bash
npm run build
```
3. 将构建后的 `dist` 目录部署到 Nginx 或其他 Web 服务器

### 4. 权限配置
1. 登录系统后台，为角色分配课程管理模块的权限
2. 重启后端服务，使新添加的菜单生效

## API 接口说明

### 课程管理接口
- `GET /course/list`：查询课程列表
- `GET /course/{id}`：查询课程详情
- `POST /course/add`：新增课程
- `PUT /course/update`：更新课程
- `DELETE /course/delete/{id}`：删除课程
- `DELETE /course/deleteBatch`：批量删除课程

### 选课管理接口
- `GET /course/selection/list`：查询选课记录列表
- `GET /course/selection/{id}`：查询选课记录详情
- `POST /course/selection/add`：新增选课记录
- `PUT /course/selection/update`：更新选课记录
- `DELETE /course/selection/delete/{id}`：删除选课记录
- `DELETE /course/selection/deleteBatch`：批量删除选课记录

### 成绩管理接口
- `GET /course/score/list`：查询成绩列表
- `GET /course/score/{id}`：查询成绩详情
- `POST /course/score/add`：新增成绩
- `PUT /course/score/update`：更新成绩
- `DELETE /course/score/delete/{id}`：删除成绩
- `DELETE /course/score/deleteBatch`：批量删除成绩

## 注意事项

1. 当前版本使用了模拟数据作为教师、学生的下拉选项，在实际部署时需要替换为真实的 API 调用
2. 需要根据实际的权限系统，为不同角色分配相应的操作权限
3. 如果需要更复杂的功能（如课程人数限制、教师评价等），可以在现有代码基础上进行扩展
